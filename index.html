React - Starter Guide


React is a JavaScript Library for building User Interfaces

React makes building user interfaces incredibly easy and intuitive

We will begin by explaining some important features of React:

Declarative

React is what we call a Declarative Library

But what do we really mean when we say React is Declarative?

Normally when we want to change elements in JavaScript we write a bunch of code to select the specific element we want to modify

  • For example, we could make HTML change the innerText of some element after the user clicks a button with the '.onclick' event
  • We could also make some message appear after the pages has loaded using the '.onload' event
  • The list of situations where we can use events is endless.
  • All of these elements are being changed with the use of events, React will instead change them in a declaratively way.

So what is this "declarative way"? Explain Further...

The HTML will know how to change itself accordingly to the specific value we give it

This is what we mean by being declarative

This implies that we we dont need to change the HTML code directly.

This makes your code predictable and easier to debug

React Components

React has a feature that we call Components

Components are the Most Important part of react

React: It's Component-Based

When we say React is Component-Based, what we actually mean is that instead of building only one giant JavaScript file, which stores a considerable amount of code, we are actually going to build smaller JavaScript files which stores less code overall and which represent only individual parts of our page.

These smaller elements will be chained together to build up our entire webpage.

This feature makes things easier to manage, before bringing them all together to make more complex UI

React changes the mindset of how you're supposed to think about programs

React makes us think less about the workflow and more about being component-based and state-based

Components don't have to be big

The same Component can have different appearences in different situations, even if it's the same component, the reason being that this component may have some other components inside of it.

example: google react page

React: Benefits

React is a library that allows us not only to write our HTML but that also allows us to do HTML interact in the way that want with our state

React allows us to manage the state inside of our application

React also allows us to manage the state and rendering of the user interface and user interaction in the website

It allows you to automatically re-render your application, everytime you state changes
